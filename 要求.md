请使用标准 C++（C++11及以上）编写一个完整的“智能储物箱管理系统”课程设计项目。

项目定位：
- 面向大一《C++程序设计》课程结课作业
- 控制台程序（不使用图形界面）
- 不涉及真实硬件，仅模拟储物箱管理
- 代码风格适合大学生课程设计答辩
- 尽量体现面向对象思想

功能要求：

一、储物箱管理

系统维护若干储物箱。

每个储物箱包含：

- 箱号
- 是否占用
- 用户姓名
- 电话号码
- 物品名称
- 取件码
- 存入时间

支持：

1. 查看所有储物箱
2. 查看空闲储物箱
3. 查看已占用储物箱

二、存物功能

用户输入：

- 姓名
- 电话
- 物品名称

系统自动寻找空闲储物箱。

若存在空闲箱：

- 自动分配箱号
- 自动生成4位随机取件码
- 记录存入时间

显示：

存物成功
箱号
取件码

若无空闲箱：

提示储物箱已满。

三、取物功能

用户输入：

- 箱号
- 取件码

验证成功后：

- 显示物品信息
- 清空储物箱
- 状态改为空闲

验证失败：

提示取件码错误。

四、查询功能

支持：

1. 按箱号查询
2. 按姓名查询
3. 按电话查询

显示完整信息。

五、管理员功能

管理员菜单：

1. 添加储物箱
2. 删除空闲储物箱
3. 查看统计信息

统计信息包括：

- 总箱数
- 已占用数量
- 空闲数量
- 使用率

六、数据持久化

程序启动时：

自动从文件读取数据。

程序退出时：

自动保存数据。

使用：

fstream

保存到：

locker.txt

七、面向对象设计要求

至少设计以下类：

Locker

成员：

- 箱号
- 状态
- 用户信息
- 物品信息
- 时间信息

方法：

- storeItem()
- takeItem()
- display()

LockerManager

成员：

- vector<Locker>

方法：

- loadData()
- saveData()
- storeItem()
- takeItem()
- searchLocker()
- addLocker()
- deleteLocker()
- showStatistics()

八、输入合法性处理

处理：

- 非法菜单输入
- 不存在的箱号
- 空姓名
- 空物品名称
- 错误取件码

避免程序崩溃。

九、代码要求

- 使用 vector
- 使用 string
- 使用 fstream
- 使用 time 库记录时间
- 使用随机数生成取件码
- 使用类和对象
- 添加充分注释
- 模块化设计
- 可直接编译运行

十、输出要求

请生成：

1. 完整项目代码
2. 文件结构说明
3. 类图（文本形式）
4. 主要函数说明
5. 运行示例
6. 课程设计报告（约1500字）
7. 项目特色与创新点说明

代码要达到课程设计提交水平，而不是教学示例水平。
