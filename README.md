# 智能储物箱管理系统 —— 课程设计报告

---

## 一、项目概述

本系统是一个基于 C++ 面向对象思想开发的"智能储物箱管理系统"控制台程序，模拟超市、图书馆等场所常见的自助储物柜功能。用户可通过菜单完成存物、取物、查询和管理等操作，所有数据自动持久化保存到本地文件。

**开发环境：** Windows 11 / g++ (C++11)  
**程序类型：** 控制台应用程序  
**代码行数：** 约 600 行（含注释）

---

## 二、文件结构说明

```
my-cc/
├── main.cpp               # 程序入口与菜单交互
├── Locker.h               # 储物箱类声明（数据成员与方法接口）
├── Locker.cpp             # 储物箱类实现（核心操作逻辑）
├── LockerManager.h        # 管理类声明（业务逻辑接口）
├── LockerManager.cpp      # 管理类实现（数据持久化与系统管理）
├── locker.txt             # 运行时自动生成的数据文件
└── 课程设计报告.md         # 本文档
```

各文件职责分明，遵循"声明与实现分离"的模块化原则。`main.cpp` 负责用户交互，`Locker` 类封装单个储物箱的数据与行为，`LockerManager` 类负责系统级别的管理与数据持久化。

## 四、主要函数说明

### 4.1 Locker 类核心方法

| 方法 | 参数 | 返回值 | 功能说明 |
|------|------|--------|----------|
| `Locker(int)` | 箱号 | — | 初始化一个空闲储物箱 |
| `storeItem(...)` | 姓名、电话、物品、取件码 | void | 记录用户信息、物品信息、当前时间，标记为已占用 |
| `takeItem()` | 无 | void | 清空所有用户数据，恢复为空闲状态 |
| `display()` | 无 | void | 格式化输出储物箱完整信息（含时间） |
| `formatTime(time_t)` | 时间戳 | string | 静态方法，将时间戳转为 "YYYY-MM-DD HH:MM:SS" 格式 |

### 4.2 LockerManager 类核心方法

| 方法 | 功能说明 |
|------|----------|
| `loadData()` | 启动时从 `locker.txt` 逐行读取，按 `\|` 分隔解析7个字段 |
| `saveData()` | 退出时将全部储物箱按相同格式写入 `locker.txt` |
| `storeItem(...)` | 查找空闲箱 → 生成4位随机码 → 调用 Locker::storeItem() |
| `takeItem(id, code)` | 验证箱号存在性 → 验证取件码 → 显示物品 → 清空 |
| `searchById(int)` | 遍历 vector，匹配箱号并调用 display() |
| `searchByName(string)` | 遍历 vector，匹配姓名（仅已占用箱） |
| `searchByPhone(string)` | 遍历 vector，匹配电话（仅已占用箱） |
| `addLocker()` | 创建新 Locker 对象，箱号自增 |
| `deleteLocker(int)` | 按箱号查找，仅空闲箱可删除 |
| `showStatistics()` | 统计总数、占用数、空闲数、使用率百分比 |
| `generateCode()` | 利用 `rand()` 生成 1000~9999 随机取件码 |

### 4.3 main.cpp 辅助函数

| 函数 | 功能说明 |
|------|----------|
| `showMainMenu()` | 显示主菜单（存物/取物/查询/查看/管理/退出） |
| `showSearchMenu()` | 显示查询子菜单（按箱号/姓名/电话） |
| `showViewMenu()` | 显示查看子菜单（所有/空闲/已占用） |
| `showAdminMenu()` | 显示管理员子菜单（添加/删除/统计） |
| `readInt(prompt)` | 带循环校验的整数输入，防止非数字输入导致程序崩溃 |
| `readNonEmptyString(prompt)` | 带循环校验的字符串输入，拒绝空输入和纯空白输入 |
| `pauseForEnter()` | 暂停屏幕，等待用户按回车后继续 |
| `clearInput()` | 清空 cin 错误状态与缓冲区 |

## 六、项目特色与创新点

### 6.1 面向对象设计完善

系统严格遵循面向对象思想，将"储物箱"抽象为独立的 `Locker` 类，"管理系统"抽象为 `LockerManager` 类。两个类职责清晰：`Locker` 只关心单个储物箱的状态与行为；`LockerManager` 关心系统的整体协调。这种设计符合"单一职责原则"，使得代码易于理解和维护。

### 6.2 RAII 自动数据管理

利用 C++ 的 RAII（Resource Acquisition Is Initialization）机制：构造函数中自动调用 `loadData()` 加载数据，析构函数中自动调用 `saveData()` 保存数据。无论程序是正常退出还是意外终止（如按 Ctrl+C），只要析构函数被执行（通过全局对象的正常析构流程），数据就不会丢失。这比手动调用 save/load 的方式更加安全和优雅。

### 6.3 健壮的输入校验

系统对所有用户输入进行了严格的合法性检查：
- **整数输入校验**：使用 `cin >> value` 的返回值判断输入是否为有效整数，失败时清空错误标志并提示重新输入，避免非数字输入导致死循环或崩溃。
- **字符串非空校验**：拒绝空字符串和纯空白字符的输入，确保姓名、电话、物品名称等关键字段不为空。
- **业务逻辑校验**：检查箱号是否存在、储物箱是否空闲、取件码是否匹配、删除时是否被占用等。

### 6.4 跨平台兼容

格式化时间时使用条件编译（`#if defined(_WIN32)`），在 Windows 上使用 `localtime_s`，在 Linux/macOS 上使用 `localtime_r`，确保代码在不同平台下均能安全编译运行。

### 6.5 用户友好的界面设计

采用 Unicode 框线绘制菜单边框和成功提示框，界面清晰美观。操作后自动暂停等待用户按回车，避免信息被滚动覆盖。子菜单支持返回上级，操作流程符合直觉。

### 6.6 数据持久化设计

使用简单的 `|` 分隔文本格式存储数据，既方便程序读写，也方便用户用文本编辑器直接查看和修改。文件名为 `locker.txt`，与程序放在同一目录下，管理方便。数据结构清晰，每行7个字段对应 `Locker` 类的完整状态。

## 七、技术要点总结

| 技术要点 | 具体应用 |
|----------|----------|
| **类与对象** | `Locker` 类封装储物箱数据与操作；`LockerManager` 类封装系统管理逻辑 |
| **vector 容器** | `vector<Locker>` 动态管理储物箱列表，支持增删改查 |
| **string 字符串** | 所有文本字段使用 `std::string` 存储 |
| **fstream 文件流** | `ifstream` / `ofstream` 实现数据持久化读写 |
| **time 时间库** | `time_t` + `localtime_s/r` + `strftime` 记录与格式化 |
| **随机数** | `rand()` + `srand(time(0))` 生成4位取件码 |
| **输入流处理** | `cin.ignore()` 清空缓冲区、`cin.clear()` 清除错误标志 |
| **条件编译** | `#if defined(_WIN32)` 处理平台差异 |
| **RAII** | 构造自动加载、析构自动保存 |
